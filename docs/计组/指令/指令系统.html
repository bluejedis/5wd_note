基本格式

    概念

    • 指令
      • 定义：一条指令 is 机器语言的一个语句
        • 本质：一组有意义的 二进制代码
      • 组成部分：
        • OP+A

    操作码 & 地址码

    • 操作码
      • 指出该指令应执行什么 操作以及具有何种 功能
        • 识别指令
        • 了解指令功能
        • 区分操作数地址内容
      • 例如：
        • 运算类型
          • 算术加运算
          • 算术减运算
        • 程序控制
          • 程序转移
          • 返回操作
    • 地址码
      • 给出被操作的信息（指令或数据）的地址
      • 包括：
        • 参加运算的一个或多个操作数的地址
        • 运算结果的保存地址
        • 程序的转移地址
        • 被调用子程序的入口地址等

    ~ 字长

    • 定义：
      • 一条指令所包含的 二进制代码的位数
    • 决定因素：
      • 操作码的长度
      • 地址码的长度、个数
    • 与机器字长的关系：
      • = 机器字长
      • or < >机器字长
    • 分类：
      • 单字长指令：1 机器字长
      • 半字长指令： 1/2机器字长
      • 双字长指令：2*机器字长

    attention:

    • 指令长度不同会影响取指令时间开销：
      • 单字长指令：
        • 只需访存1次
        • 能将指令完整取出
        • 耗费1个存取周期
      • 双字长指令：
        • 需要访存2次
        • 才能完整取出指令
        • 耗费2个存取周期

    指令字结构

    • 定长指~：
      • 所有指令的长度都相等
      • 特点：执行速度快，控制简单
    • 变长~：
      • 各种指令的长度随指令功能而异
    • 注意：因为主存一般是按字节编址的，所以指令字长通常为字节的整数倍

    pro：根据指令格式及相关编码条件组合成机器代码（2015）

    指令格式 分类

      零地址指令

      • only 操作码OP，没有显式地址
      • 两种可能：
        • 不需要操作数的指令
          • 如空操作指令、停机指令、关中断指令等
        • 零地址的运算类指令（用在堆栈计算机中）
          • 参与运算的两个操作数从栈顶和次栈顶弹出
          • 送到运算器进行运算
          • 运算结果压入堆栈

      一~

      • 两种常见形态：

        • 只有目的操作数的单操作数指令：
          • 执行过程：
            • 按A₁地址读取操作数
            • 进行OP操作后，结果存回原地址
          • 指令含义：
            • OP(A₁)→A₁
          • 典型指令：
            • 加1
            • 减1
            • 求反
            • 求补
            • 移位

        • 隐含约定目的地址的 双~指令：
          • 执行过程：
            • 按指令地址A₁读取源操作数
            • ACC（累加器）提供另一个操作数
            • 运算结果存放在ACC中
          • 指令含义：
            • (ACC)OP(A₁)→ACC

      pro：地址位数与寻址范围的关系（2010、2021）

      寻址范围
      • 若指令字长为32位：
        • 操作码占8位
        • 1个地址码字段占24位
        • 操作数的直接寻址范围为2²⁴=16M
      • 访存次数：
        • 若地址码字段均为主存地址
        • 需要3次访存：
          • 取指令1次
          • 取操作数1次
          • 存结果1次

      二~

      • 指令含义：(A₁)OP(A₂)→A₁
      • 使用场景：
        • 常用的算术和逻辑运算指令
        • 需要 2* 操作数
        • 目的操作数地址用于保存运算结果
      寻址范围
      • 若指令字长为32位：
        • 操作码占8位
        • 两个地址码字段各占12位
        • 每个操作数的直接寻址范围为2¹²=4K
      • 访存次数：
        • 若地址码字段均为主存地址
        • 需要4次访存：
          • 取指令1次
          • 取两个操作数 2次
          • 存结果1次

      三~

      • 指令含义：(A₁)OP(A₂)→A₃
      寻址范围
      • 若指令字长为32位：
        • 操作码占8位
        • 3个地址码字段各占8位
        • 每个操作数的直接寻址范围为2⁸=256
      • 访存次数：
        • 若地址码字段均为主存地址
        • 需要4次访存：
          • 取指令1次
          • 取两个操作数2次
          • 存结果1次

      四~

      • 指令含义：
        • (A₁)OP(A₂)→A₃
        • A₄=下一条将要执行指令的地址
      寻址范围
      • 若指令字长为32位：
        • 操作码占8位
        • 4个地址码字段各占6位
        • 每个操作数的直接寻址范围为2⁶=64
      • 访存次数：
        • 若地址码字段均为主存地址
        • 需要4次访存：
          • 取指令1次
          • 取两个操作数2次
          • 存结果1次

扩展 操作码指令格式

pro：扩展操作码的设计与分析（2017、2021、2022）

    基本概念

    • 设计目的：
      • 在指令字长有限的前提下保持丰富的 指令种类
    • 特点：
      • 操作码字段 位数 不fix
      • 分散放在 指令字的不同位置上
      • 操作码 长度 随地址码的减少而增加

    具体实现

    操作码的位数 随 地址数的减少而增加

    • 示例分析（16位指令字长）：
      • 基本结构：
        • 4位基本操作码字段OP
        • 3个4位长的地址字段A₁、A₂、A₃
      • 指令分配：
        • 三地址指令：15条（1111留作扩展操作码）← 剩的那条
        • 二地址指令：15条（11111111留作~）
        • 一地址指令：15条（111111111111留作~）
        • 零地址指令：16条

    注意

    • 关键原则：
      • 短码 不能是长码的前缀
      • 各指令的操作码不能重复
    • 优化建议：
      • 使用fff高的指令分配较 短操作码
      • 使用fff 低的指令分配较长操作码

寻址方式

  • 寻址方式：
    • 定义：寻找指令或操作数有效地址的方式
    • 作用：确定
      • 本条指令的数据地址
      • 下一条待执行指令的地址
    • 分类：
      • 指令寻址
      • 数据寻址

    指令寻址和数据寻址

    • 指令寻址：寻找 下一条将要执行的指令地址
    • 数据寻址：寻找 本条指令的数据~

      指令寻址

        顺序 寻址

        • 通过 PC 自增实现
        • ~过程
          • PC+1条指令的 长度
            • 自动形成下一条指令的地址
          • 大小
            • 与 编址方式、指令字长有关

        pro：PC自增大小与编址方式、指令字长的关系（2013、2014、2019、2023）

        attention:

        • 现代计算机通常是按字节编址的
        • 指令字长：
          • 16位指令字长
            • PC自增为 (PC)+2(\mathrm{PC})+\color{black}2(PC)+2
          • 32位指令字长

        跳跃寻址

        通过转移类指令实现

        • 定义
          • 由本条指令 给出下条指令地址的计算方式

            • 地址码 直接指出转移目标地址

          • 地址码指出转移目的地址 相对于当前PC值的偏移量
        • 转移指令执行结果
          • 修改PC值
          • CPU根据PC的内容去主存取下一条指令

      基本 寻址方式

        隐含~

        不明显地给出操作数的地址，而是隐含操作数的地址

        • 单地址指令格式的隐含约定

          • 第二个操作数由累加器(ACC)提供
          • 指令中只明显指出第一个操作数的地址

            图4.2隐含寻址

        • 优缺点

          • 优点：有利于缩短指令字长
          • 缺点：需增加存储操作数或隐含地址的硬件

        立即（数）~

        pro：立即寻址的概念（2023）

        • 概念
          • 指令字中的地址字段
            • 不是操作数的地址
            • 而是 操作数 本身
              • 也称立即数
              • 采用补码表示
          • #: 立即寻址 特征
          • A: 操作数
        • 优缺点
          • 优点
            • 指令在执行阶段不 访 存
            • 指令执行速度最 快
          • 缺点
            • A的位数 限制了立即数的范围

        直接~

        • 指令字中的形式地址A就是操作数的 真实地址EA
          • EA = A

        • 优缺点
          • 优点
            • 简单，不需要专门计算操作数的地址
            • 指令在执行阶段仅需访存 一次
          • 缺点
            • A的位数 限制了该指令操作数的寻址范围
            • 操作数的地址不易修改

      间接 寻址类

        间接~

        • 指令的地址字段给出的是操作数有效地址所在主存单元的地址
          • 即操作数地址的地址
          • EA = (A)


        图4.5间接寻址

        • 优缺点
          • 优点
            • 可扩大寻址范围
            • 便于编制程序
          • 缺点：指令在执行阶段要多次访存

        R 寻址

        • 与直接寻址原理相同，但访问寄存器而非主存
          • 指令的地址字段给出操作数所在寄存器的编号
          • EA = Ri

        pro：寄存器编号位数与寄存器数量的关系（2022）

        • 优缺点
          • 优点
            • 执行速度快，不用访存
            • 指令字长较短
          • 缺点
            • 寄存器价格昂贵
            • CPU寄存器数量有限

        • 综合了间接寻址和寄存器寻址特点
          • Ri所指寄存器给出操作数所在主存单元的地址
          • EA = (Ri)

        • 特点
          • 面向 OS
          • 内容由操作系统或管理程序确定
          • BR内容不变
        • 优点
          • 可以 扩大寻址 范围
          • 有利于多道程序设计
          • 可用于编制浮动程序
        • 缺点：偏移量 位数较 短

        pro：变址寻址的EA的计算（2013），先变址后间址方式的EA的计算（2016）

        • EA = (IX) + A
        • IX+指令字中的形式地址A
        • IX: 变址寄存器
          • 可用 专用变址寄存器或通用寄存器

        pro：变址寻址的适用场景（2017）

        • 特点
          • 面向 user
          • 执行过程中变址寄存器内容可变
          • 形式地址A不变

        pro：变址寻址访问数组的过程（2018）

        • 与基址寻址的区别
          • 基址寻址
            • 面向系统
            • 基址寄存器内容不可变
            • 指令字中A可变
          • 变址寻址
            • 面向用户
            • 变址寄存器 内容 可变
            • 指令字中A不可变

        pro：偏移寻址的范畴（2011）

          • 相对寻址
          • 基址寻址
          • 变址寻址
